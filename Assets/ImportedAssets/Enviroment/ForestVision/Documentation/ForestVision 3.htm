ForestVision allows you to create and use an unlimited library of vegetation assets easily!

Getting Started: URP

 

URP Requirements: 2020.3.10f1 +

To set ForestVision up for URP:

        Load an empty URP project with the above version of Unity import ForestVision

        And then in the ForestVision/Packages folder, extract the FV3_URP package.

Thats it!

Note, it will immediately overwrite all the default FV materials with the URP versions.

 

One final thing you can do is to set the Quality preset, which basically just sets up an Ultra quality.

 

 

Getting Started

All ForestVision assets can be found by accessing the FV menu from GameObject/ 3D Object/ FV either from the main menu, or by right clicking in the Hierarchy window.

As you can see, FV is now comprised of core Assets. Each of these assets (except for the Woods Background) have several mesh options to choose from, and most also can cycle through texture options as well. The Starter Kit quickly allows you to add some base assets to your scene for editing.

 

Branches

To see this in action, add a FV/ Branch Mesh to your scene, then load up the Switch Mesh Tool by going to the main menu: Tools/ForestVision/FV Tools/ Switch Mesh. Now select on your branch mesh and use the sliders to adjust your branch asset.

In this example, the Branch asset is unique in that it can switch to no foliage, foliage, and increased foliage. Of course, it also has 8 texture options as well!

 

Cards

Cards are useful when you are building your own trees to create the actual foliage

 

Grass

Grass assets do not have texture options, but they do have 20 different meshes!

High Resolution Plants

These assets have a very high poly count so be aware of that if you want to use them in a mobile game

 

Rocks

This asset has 4 built in textures and 11 mesh options.

 

 

Specific Tools

ForestVision comes with a handful of tools that I have found to be useful and wanted to make them available.

 

Switch Mesh

As it sounds, you can switch the mesh of FV assets with this tool.

Found by going to the main menu: Tools/ ForestVision / Tools / Switch Mesh

Switch Mesh, and almost all FV tools, are selection dependent, once you have a selection, IF what you are selected on is a FV asset, then your options will be displayed.

Switch Species is your texture. Each asset is different in the options available to it with Switch Mesh, but the Species always refers to the specific texture applied to the mesh. For example, trunks have 32 mesh options, and 8 species (birch, oak, pine, etc.)

Include Children typically will remain selected, but there might be a time when you only want to update the texture on what is actively selected, and not affect its children.

In the example below, you can see the Switch Mesh tool in action on a new branch being created. I added a single branch asset, then duplicated it twice, then made those 2 copies children of the first branch. Now whenever I move the original, the other 2 move as well.

Note: youll see that the Switch Mesh Slider will only ever affect the current selection. This is to preserve any custom settings applied to its children.

 

A specific set of options available to Branches are its Foliage (Bare, Basic Foliage, Max Foliage)

Here you can see a branch with Basic Foliage, with the option to Remove or Increase its amount. Removing its Foliage obviously will turn the branch completely bare. You can only Add Foliage to a Bare branch. Once a branch as foliage, you get an additional option to Increase its foliage. This foliage setting is independent of the current species setting.

 

 

Swap Material

The beauty of this tool is that it examines your selection and will apply a desired material only to what makes sense.

In the example below, you see I am selected on a branch that has 2 additional branches in its hierarchy. However, the root object that is selected has a Bark Material applied. Using the Swap Material toolset to swap to a Foliage Material, only the Foliage materials of every child in the selection hierarchy get updated. It leaves the Bark materials intact.

So, if you chose to change out the Bark material instead of Foliage, only the Bark materials would change.

Also worth noting, each Foliage and Bark material FV comes with offers you another 8 textures to use!

 

New Optimized

Want to take your new tree, or bush, or whatever FV creation you came up with, and collapse all those connected prefabs down into a single optimized asset? Then this is your tool. Below you can see one of our basic prefabs, which contains quite a few sub-meshes. As cool as this tree is with its ability to be continually customized and altered to fit a specific visual requirement, its not very game friendly.

 

Load up the Optimize Tool: Tools/ForestVision/Tools/New Optimized

Right off the bat you can see what the final vertex count of your mesh will be once optimized. Remember, this isnt an optimizer that refines vertices, it refines your hierarchy. The point here is that if you are going for a specific vertex threshold, you can use this count to let you know where you stand.

 

If you want to change the name, or set a specific path, you have those options. If you dont make any changes, then you can see the name of the new mesh will have Opt_ prefixed to the current name and it will get dumped into the Optimized folder.

 

Collapse to single Mesh button will run through your selection and do a couple of things:

1.      Creates a new single mesh object from your current hierarchy.

2.      Reapplies your materials so that your Bark and Foliage Materials arent lost.

3.      Adds a simple LOD Group component with culling at 10%. Feel free to adjust as you wish.

4.      Adds a FV_Items script set to a category of Optimized, and provides a basic name

a.      Note, if your current selection already had FV_Items assigned, then those values will carry over.

5.      Finally, it creates a new prefab and saves it to the ForestVision/Optimized folder along with the new mesh.

 

New Prefab

Save your current creation as a unique prefab very quickly with this tool.

The great thing about this pipeline is you can very easily and quickly make a ton of different varieties of your creations.

When you save the new prefab, it will also add a FV_Items script and default it to Trees. Feel free to update that you whatever you want.

FV Items

Youve heard me talk a lot about this script so what is it?

Basically, it is connected to the Browser and allows you to tag your assets and have them organized for the Browser to view.

Backgrounds, Branches, Flora, Ground, LowPoly, Mobile, Optimized, Specials, Trees, Trunks, and Vines are the categories available.

Its very straightforward, apply this script to any asset that you want to show up in the Browser, set the Category and Name and youre done.

Note, it will need to be saved as a Prefab somewhere inside of the ForestVision hierarchy for the Browser to find it.

 

Browser

Finally, the Browser, a cool idea from long ago that ended up having some issues with Unity version upgrades, and some bugs that were fixed, but ultimately became more of a headache and kept me from creating assets for FV.

Regardless, it does work, Im just not going to support any issues that may arise from it.

The basic premise of the Browser is to organize your assets into one easy place for viewing and access. As you create new trees, bushes, whatever, tag those assets and let the browser arrange them for you.

To use it, Tools / ForestVision / Deprecated / Browser

Once it opens, it will scan the FV folder for assets that have the FV Items script attached, generate thumbnails, and then sort them into their categories. Use the category tabs to quickly jump to what you are looking for and then just click on the thumbnail to add that asset to your scene.

Note, if nothing is selected, it will add the asset to world center, otherwise, it will add it as a child of what you have selected.

 

Refresh Thumbnails button will force a refresh of all the thumbnails in the browser, which can happen if you have the browser opened, and you add a new FV Item to a folder, it will not scan for changes automatically, so hit this button to get the update.

 

BUGS: yes there are still some running around in the browser, but worse case, if it starts giving you a fit, refresh the thumbnails, or as a final resort just close and reopen it.

 

 

FAQ

The most asked question I get about ForestVision is if it supports URP and HDRP.

The answer is yes.

I have optimized the assets in FV to the point that they have a single mesh and a single material. Prior to version 3.0, a single tree could be comprised of several meshes, with several materials assigned. That made for a very large headache when it came to trying to create HDRP, or URP materials for specific prefabs. By condensing everything down the way I have, it is incredibly easier now for the user to create a new material in whatever rendering pipeline you choose and apply the textures supplied.

For example, in version 3.0 there are only 3 Foliage materials and 6 barks. Before there was about 75 barks and 100+ foliage. Now that the 3.0 redesign is complete, I will continue building out new prefab trees for ease of use and will compile URP and HDRP packages with those specific materials embedded, but in the meantime, you can still use FV in each render pipeline, just recreate the materials and drag the 2 textures (diffuse and normal) to your channels and youre done.
